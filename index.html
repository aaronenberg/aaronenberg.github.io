Recursive Descent Parser

beep Grammar for Arithmetic Expression
EXP ::= EXP + TERM | EXP - TERM | TERM
TERM ::= TERM * FACTOR | TERM / FACTOR | FACTOR
FACTOR ::= ( EXP ) | DIGIT
DIGIT ::= 0 | 1 | 2|3

Example of a valid set:
3-2/(1+3)$
Example of an invalid set:
1+/2

Input a valid arithmetic expression (like we learned in elementary) containing only 0-3, the arithmetic operators +, -, *, /, and parentheses to surround valid expressions (, ) which can be joined by the operators.
Terminate the input stream with a single dollar sign $.
Clicking 'submit', will run the parser to check whether the input is accepted by the grammar above.

PL Assignment 1

(a) Syntax

- A set of rules that defines the combination of symbols that are considered to be a correctly structured document or fragment in a language (Wikipedia).
- The aspects of a programming language that can be modeled by a context-free grammer (Linz).


(b) Semantics

- The rules of a programming language that determine how the meaning of a particualar construct is interpreted (Linz).


(c) Lexical Scanning

- The process of converting a sequence of characters into a sequence of tokens, or strings with certain semantic meaning (Wikipedia).


(d) Parsing and Parser

- Parsing, or syntax analysis, is a way of describing the structure of a sentence through its grammatical derivation (Linz).
- A recursiveDescentParser is a software component that takes input data, usually text, and builds a hierarchical data structure, giving a structural representation of the input and also checking for correct syntax in the process (Wikipedia).
